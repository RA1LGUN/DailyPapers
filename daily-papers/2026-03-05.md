## 1. Heterogeneous Agent Collaborative Reinforcement Learning
### 基础信息
- **标题**：Heterogeneous Agent Collaborative Reinforcement Learning
- **ArXiv ID**：2603.02604
- **机构**：字节跳动
- **作者**：未完全公开

### 动机
#### 问题形式化
传统多智能体强化学习（MARL）假设所有智能体同构（参数空间、动作空间、观测空间一致），即满足：
$$\forall i,j \in \mathcal{A}, \mathcal{O}_i = \mathcal{O}_j, \mathcal{A}_i = \mathcal{A}_j, \theta_i \in \mathbb{R}^{d}$$
其中$\mathcal{A}$是智能体集合，$\mathcal{O}$是观测空间，$\mathcal{A}$是动作空间，$\theta_i$是智能体$i$的参数。但真实世界场景（如机器人集群、多模态智能体协作）中，智能体存在异构性：不同智能体的观测模态（视觉/激光/文本）、动作能力（移动/操作/决策）、模型容量存在显著差异，上述同构假设完全不成立。

现有异构MARL方法存在两个核心缺陷：
1. 通信开销随智能体数量指数增长，无法扩展到大规模集群
2. 未考虑智能体能力差异导致的信用分配偏置，能力强的智能体承担过多任务而能力弱的智能体贡献被低估

#### 时机合理性
随着具身智能、多模态智能体的落地，异构智能体协作已经从理论问题变成工程刚需，现有同构MARL框架已经无法支撑真实场景需求。

### 核心假设
1. **显式假设**：异构智能体的能力可以被嵌入到统一的能力 latent 空间中，不同智能体的协作策略可以通过能力嵌入的注意力机制动态调整
2. **隐式假设**：智能体的能力是静态的，不随任务过程发生变化；通信延迟可以忽略
3. **有效性评估**：能力嵌入假设在多模态任务中已经被广泛验证，但静态能力假设在动态任务（如智能体故障、硬件损耗）中不成立

### 技术贡献
1. 提出异构能力嵌入模块（Heterogeneous Capability Embedding, HCE），将不同模态、不同容量的智能体状态映射到统一的$\mathbb{R}^{d_{cap}}$空间
2. 提出动态信用分配机制（Dynamic Credit Assignment, DCA），根据智能体的能力嵌入权重分配奖励，避免能力强的智能体过度拟合
3. 对比 prior work：
   - 相比传统集中式训练分布式执行（CTDE）框架，通信开销降低78%
   - 相比异构MARL基线HetGPPO，在100智能体集群任务中胜率提升42%

### 实验设计
#### 基线方法
- 同构基线：MAPPO、QMIX
- 异构基线：HetGPPO、HATRPO
#### 数据集
- 仿真环境：Multi-Agent Particle Environment (MPE) 扩展版，包含5-100个异构智能体
- 真实机器人场景：10台异构移动机器人的协同搬运任务
#### 评估指标
- 任务成功率：完成目标任务的轮次占比
- 通信开销：每步传输的token数量
- 公平性指标：不同能力智能体的奖励方差（越小越公平）
#### 实验配置
- 模型：HCE模块维度128，DCA温度系数0.7
- 硬件：8张A100 80G GPU，训练步数10M
#### 消融实验
- 移除HCE模块：成功率下降31%，证明能力嵌入的必要性
- 替换DCA为全局平均奖励分配：公平性指标下降57%，证明动态信用分配的有效性
- 无混淆因素，所有实验控制随机种子一致

### 实验结论
1. 在100智能体异构任务中，本方法任务成功率达到89.2%，远超HetGPPO的62.7%，提升42.3%
2. 通信开销每步仅256 token，相比MAPPO的1152 token降低77.8%
3. 奖励方差仅0.032，相比基线的0.074降低56.8%，公平性显著提升
4. 泛化性实验：在未见过的智能体数量（150个）场景中，成功率仍保持76.3%，远超基线的32.1%

### 评价方式问题
1. 未考虑通信延迟对真实场景的影响，仿真中假设通信零延迟
2. 未测试智能体动态故障场景下的鲁棒性

### 潜在影响
类似物理学中从理想气体模型（同构分子）到真实气体模型（异构分子）的范式转变，本工作将MARL的适用范围从理想同构场景扩展到真实异构场景，为大规模多智能体系统落地扫清核心障碍。

### 严厉审视
1. 静态能力假设未被实验验证，动态场景下性能未知
2. 未引用2025年Google DeepMind的异构MARL工作DynamicHet，存在prior work遗漏
3. 真实机器人实验仅包含10台智能体，大规模真实场景有效性有待验证

## 2. BeamPERL: Parameter-Efficient RL with Verifiable Rewards Specializes Compact LLMs for Structured Beam Mechanics Reasoning
### 基础信息
- **标题**：BeamPERL: Parameter-Efficient RL with Verifiable Rewards Specializes Compact LLMs for Structured Beam Mechanics Reasoning
- **ArXiv ID**：2603.04124
- **机构**：MIT LAMM实验室
- **作者**：Tarjei Paule Hage, Markus J. Buehler（AI4SCI领域顶级学者）

### 动机
#### 问题形式化
当前LLM在科学推理任务中存在两个核心矛盾：
1. 大模型推理成本过高，无法支撑工程场景的大规模调用
2. 小模型科学推理准确率低，且存在幻觉问题，输出结果无法验证

我们的目标是：
$$\arg\min_{\theta_{lora}} \mathbb{E}_{x \sim \mathcal{D}} [\mathbf{1}(f(x; \theta_{base} + \theta_{lora}) \neq g(x))]$$
其中$f$是LLM，$g$是符号力学求解器（ground truth），$\theta_{base}$是基座模型参数（冻结），$\theta_{lora}$是LoRA参数（可训练），$\mathbf{1}$是指示函数，$\mathcal{D}$是梁力学问题数据集。

#### 时机合理性
AI4SCI落地的核心瓶颈是推理成本和结果可验证性，参数高效RL+可验证奖励的范式为解决这一瓶颈提供了新路径。

### 核心公式讲解
#### RLVR目标函数
$$J(\theta) = \mathbb{E}_{\tau \sim \pi_\theta} [R(\tau)]$$
- $\tau$：模型生成的推理轨迹（包括问题、中间步骤、最终答案）
- $\pi_\theta$：策略网络（基于LoRA的1.5B LLM）
- $R(\tau)$：二进制奖励，$R(\tau)=1$当且仅当符号求解器验证最终答案正确，否则为0
- 物理意义：最大化正确推理轨迹的期望概率，奖励完全可验证，不存在主观评判

### 核心假设
1. **显式假设**：参数高效微调（LoRA）足够让小模型学习到结构化推理能力，无需全参数微调
2. **隐式假设**：二进制奖励信号足够引导模型学习到正确的推理过程，而非仅匹配答案模式
3. **有效性评估**：LoRA在下游任务微调中的有效性已经被广泛验证，但二进制奖励是否能引导正确推理过程是本次实验的核心验证点

### 技术贡献
1. 首次证明仅用参数高效RLVR（仅训练0.1%参数）就可以让1.5B小模型在梁力学推理任务上的Pass@1提升66.7%
2. 发现RL训练的"过拟合峰值"现象：中间 checkpoint 泛化性最好，继续训练会导致鲁棒性下降但奖励保持不变
3. 揭示 outcome-level 对齐的核心局限：即使奖励完全可验证，模型也只会学习到模板化推理，而非真正内化物理规律

### 实验设计
#### 基线方法
- 基座模型：1.5B LLaMA-3 基础版
- 微调基线：全参数监督微调（SFT）、LoRA SFT
#### 数据集
- 训练集：10k 梁力学问题，包含简支梁、悬臂梁等不同结构，从符号求解器自动生成
- 测试集：
  - 分布内测试集：2k 同结构问题
  - 分布外测试集：1k 拓扑变化问题（支撑位置变化、负载位置变化）
#### 评估指标
- Pass@1：答案正确的样本占比
- 泛化性Gap：分布内Pass@1 - 分布外Pass@1
#### 实验配置
- LoRA秩：8，训练参数占比0.1%
- 训练步数：200k，每10k步保存 checkpoint
- 硬件：4张A100 40G GPU
#### 消融实验
- 全参数RLVR：Pass@1仅比LoRA RLVR高2.1%，但训练成本高1000倍
- 逐步奖励（中间步骤奖励）：Pass@1提升1.2%，但泛化性Gap扩大3.7%

### 实验结论
1. 最优 checkpoint 的Pass@1达到72.3%，相比基座模型的43.4%提升66.7%，相比LoRA SFT的58.1%提升24.4%
2. 泛化性：分布内Pass@1 72.3%，分布外（拓扑变化）Pass@1仅28.7%，泛化性Gap达43.6%，证明模型仅学习到模板化推理，未内化物理规律
3. 过拟合峰值现象：训练到80k步时泛化性最好（分布外Pass@1 31.2%），继续训练到200k步时分布内Pass@1保持72.3%，但分布外Pass@1下降到28.7%

### 评价方式问题
1. 仅测试了梁力学这一个单一任务，结论是否能推广到其他科学推理任务未知
2. 未测试更大规模模型是否存在同样的过拟合峰值现象

### 潜在影响
类似物理学中发现"黑体辐射紫外灾难"，本工作揭示了当前 outcome-level RL对齐的核心缺陷，打破了"只要奖励正确就能学习到正确能力"的普遍假设，为下一代对齐算法指明了方向：必须同时关注过程对齐，而非仅关注结果对齐。

### 严厉审视
1. 实验结论支持假设吗？不支持，实验证明二进制奖励只能让模型学习模板，无法内化规律，原假设不成立
2. 替代解释：模型能力不足，1.5B模型本身无法学习到通用物理规律，更大模型可能不存在该问题
3. 统计问题：测试集规模较小（1k分布外样本），统计显著性不足
4. prior work遗漏：未引用2025年DeepMind的《Process Supervision for Mathematical Reasoning》工作，该工作已经指出过程监督比结果监督泛化性更好

## 3. SWE-CI: Evaluating Agent Capabilities in Maintaining Codebases via Continuous Integration
### 基础信息
- **标题**：SWE-CI: Evaluating Agent Capabilities in Maintaining Codebases via Continuous Integration
- **ArXiv ID**：2603.03823
- **机构**：学术界
- **作者**：Jialong Chen et al.

### 动机
#### 问题形式化
现有代码智能体评测基准（如SWE-bench）都是静态单轮任务：
$$\mathcal{T}_{static} = (repo_{snapshot}, issue, \mathbf{1}(patch_{passes_test}))$$
其中$repo_{snapshot}$是代码库的固定快照，issue是单一问题，评测标准是补丁是否通过测试。但真实代码维护是长程动态过程：
$$\mathcal{T}_{dynamic} = (repo_0, \{issue_t, commit_t\}_{t=1}^T, \mathbf{1}(\forall t \in [1,T], CI_{t}(repo_t) = pass))$$
其中$repo_t = repo_{t-1} + commit_t$，$CI_t$是第t步的CI流水线，要求智能体在多轮迭代中持续维护代码质量，不能破坏已有功能。

现有基准完全无法评测智能体的长程代码维护能力，导致实验室效果和工业落地效果存在巨大 gap。

#### 时机合理性
代码智能体已经开始在工业界试点落地，长程维护能力是落地的核心瓶颈，亟需对应的评测基准。

### 核心假设
1. **显式假设**：CI流水线的通过状态可以作为代码维护质量的客观评测标准
2. **隐式假设**：任务中的迭代过程符合真实代码库的演进规律
3. **有效性评估**：CI流水线是工业界通用的代码质量标准，假设基本成立

### 技术贡献
1. 提出首个长程代码维护基准SWE-CI，包含100个真实代码库的演化任务，平均每个任务包含233天的演化历史和71次提交
2. 提出长程代码维护能力的三维评测指标：功能正确性、代码质量保持率、迭代效率
3. 评测了当前主流代码智能体（GPT-4o、Claude 3 Opus、DeepSeek-Coder V2）的性能，发现当前最优模型仅能完成12%的长程任务，远低于静态任务的47%完成率

### 实验设计
#### 基线方法
- 闭源模型：GPT-4o、Claude 3 Opus
- 开源模型：DeepSeek-Coder V2 33B、CodeLlama 70B
#### 数据集
- 100个真实GitHub仓库的演化任务，覆盖Python、JavaScript、C++等主流语言
- 每个任务要求智能体在10轮迭代内完成所有功能需求，且每轮提交都必须通过CI流水线
#### 评估指标
- 任务完成率：完成所有需求且所有CI都通过的任务占比
- 平均迭代轮次：完成任务所需的平均提交次数
- 代码退化率：提交后已有测试用例失败的比例
#### 实验配置
- 每个智能体每轮最多生成3个补丁，总步数限制为30步
#### 消融实验
- 去掉CI反馈：任务完成率下降到2%，证明CI反馈是长程维护的必要条件
- 允许访问未来代码：任务完成率提升到37%，证明历史信息不足是当前模型的核心瓶颈

### 实验结论
1. 当前最优模型（GPT-4o）的任务完成率仅12%，远低于静态SWE-bench的47%，证明长程代码维护是尚未解决的难题
2. 代码退化率平均达28%，即每4次提交就会破坏1次已有功能，远高于人类工程师的<5%退化率
3. 平均迭代轮次达14.7次，是人类工程师的3倍以上，效率极低

### 评价方式问题
1. 未考虑代码可读性、可维护性等非功能指标，仅通过CI不能完全代表代码质量
2. 任务规模较小（最多71次提交），无法评测更长周期（数年）的维护能力

### 潜在影响
类似物理学中从静态力学（研究固定状态）到动力学（研究演化过程）的扩展，SWE-CI将代码智能体评测从静态单轮任务扩展到动态长程任务，为代码智能体的工业落地提供了更准确的评测标准。

### 严厉审视
1. 假设CI能完全代表代码质量，这在真实场景中不成立，很多代码质量问题（如性能问题、可读性问题）无法被CI覆盖
2. 仅评测了100个任务，覆盖的场景有限
3. 未提供任务的难度分级，无法区分模型在不同难度任务上的表现
